# Перечень автоматизируемых сценариев
## 3 сценария перехода с главной страницы сайта на страницу Курс Тестировщик ПО
### 1. Открытие страницы курса Тестировщик ПО через меню Каталог курсов
1. Открываем сайт
2. Нажимаем на Каталог курсов
3. В выпадающем меню нажимаем Программирование
4. В появившемся списке курсов находим и нажимаем на меню Тестировщик ПО
5. Открывается страница курса Тестировщик ПО
### 2. Открытие страницы курса через поиск в каталоге курсов
1. Открываем сайт
2. Нажимаем на Каталог курсов
3. В выпавшем списке находим и нажимаем последний пункт Полный каталог
4. На странице Полного каталога вводим в поле поиска Тестировщик ПО
5. Нажимаем на результат поиска - меню Тестировщик ПО
6. Открывается страница курса Тестировщик ПО
### 3. Открытие страницы через страницу Программирование
1. Открываем сайт
2. На главной странице нажимаем кнопку Программирование в каталоге Направление обучения
3. В меню страницы Программирование нажимаем на Тестировщик ПО
4. Открывается страница курса Тестировщик ПО

## 3 сценария активации экрана записи на курс со страницы Курс Тестировщик ПО
### 1. Нажатие кнопки Записаться
1. Открываем страницу курса с помощью одного из 3 сценариев открытия страницы курса
2. Нажимаем кнопку Записаться сразу после краткого описания курса
3. Переходим на экран записи на курс
### 2. Переход на экран с помощью прокрутки экрана
1. Открываем страницу курса с помощью одного из 3 сценариев открытия страницы курса
2. Прокручиваем страницу курса до конца
3. Открывается экран записи на курс
### 3. Переход на экран с помощью кнопки Записаться в верху экрана
1. Открываем страницу курса с помощью одного из 3 сценариев открытия страницы курса
2. Прокручиваем страницу вниз до появления кнопки Записаться вверху экрана
3. Нажимаем на кнопку записаться
4. Переходим на экран записи на курс

## Автоматизация отправки данных для записи и получение сообщения об отправке/не отправке данных
### 1. В случае корректных данных/пользователь авторизован/аккаунт есть
1. Активируем экран записи на курс с помощью одного из 3-х сценариев
2. Вводим данные действующего аккаунта (данные(имя, email, телефон) предоставляются перед тестированием)
3. Нажимаем кнопку записаться
4. Выдается сообщение об успешности записи 
### 2. В случае корректных данных/пользователь не авторизован/аккаунт есть
1. Активируем экран записи на курс с помощью одного из 3-х сценариев
2. Вводим данные действующего аккаунта (данные(имя, email, телефон) предоставляются перед тестированием)
3. Нажимаем кнопку записаться
4. Выдается сообщение об успешности записи
### 3. В случае корректных данных/пользователь не авторизован/аккаунта нет
1. Активируем экран записи на курс с помощью одного из 3-х сценариев
2. Вводим данные, незарегистрированные в системе, (данные(имя, email, телефон) предоставляются перед тестированием)
3. Нажимаем кнопку записаться
4. Выдается сообщение об успешности записи
### 4. Повторная отправка заявки
1. Отправляем заявку одним из перечисленных способов
2. После получения сообщения об успешности записи нажимаем кнопку Записаться повторно
3. Получаем сообщение об успешной корректировки заявки (точное сообщение уточнить)
### 5. Повторная отправка откорректированной заявки
1. Отправляем заявку одним из перечисленных способов
2. После получения сообщения об успешности корректируем данные (вводим другое имя, телефон, email)
3. Получаем сообщение об успешной корректировки заявки (точное сообщение уточнить; в случае других email\телефона возможна регистрация новой заявки)
### 6. Отправка пустой формы
1. Активируем экран записи на курс с помощью одного из 3-х сценариев
2. Нажимаем кнопку Записаться
3. Получаем сообщени об ошибке (Точное сообщение уточнить)
### 7. Отправка не своих данных/пользователь авторизован/аккаунт есть
1. Активируем экран записи на курс с помощью одного из 3-х сценариев
2. Вводим данные, незарегистрированные в системе, (данные(имя, email, телефон) предоставляются перед тестированием)
3. Получаем сообщение об успешной отправке заявки (точное сообщение уточнить)
## Автоматизация чек-листа проверки правильности заполнения каждого из полей с негативными данными
### 1. Проверка поля Имя
1. Переход к заполнению поля
2. Ввод некорректных данных (варианты проверок: "111", "№;%", "S", "qq&aa", "фыва0ячс", "")
3. Поле подсвечивается красным, выдается сообщение-подсказка под полем 
### 2. Проверка поля Телефон
1. Переход к заполнению поля
2. Ввод некорректных данных (варианты проверок: "111", "№;%", "111AAAA", "123-123-14141414", "")
3. Поле подсвечивается красным, выдается сообщение-подсказка под полем
### 3. Проверка поля Электронный адрес
1. Переход к заполнению поля
2. Ввод некорректных данных (варианты проверок: "111", "111@zz", "bbb.ry", "rrr.ty@cc", "")
3. Поле подсвечивается красным, выдается сообщение-подсказка под полем

# Перечень используемых инструментов с обоснованием выбора
* IntelliJ IDEA - среда разработки в которой будут писаться тесты.
* Java - язык программирования на котором будут написаны тесты.
* Gradle - система автоматической сборки, для создания и сборки проекта. 
* JUnit Jupiter - фреймворк для создания автотестов.
* Selenide - фреймворк-дополнение к JUnit Jupiter облегчающий написание тестов.
* Git - место для хранения и обновления тестов
* Браузеры типа Chrome и Safari - как основные браузеры на которых данный функционал должен работать

# Перечень необходимых разрешений/данных/доступов
* Доступ к тестовому стенду(сайт/API/база данных, в которой хранятся данные аккаунтов) на котором будет проводится тестирование
* Должен быть один действующий аккаунт на сайте с которого можно будет произвести запись на курс
* Должна быть возможность сделать запись на курс не имея аккаунта на сайте

# Перечень и описание возможных рисков при автоматизации
* Содержимое главной страницы может меняться
* Содержимое курса или требования к предоставленной информации может измениться (например может добавится необходимость ввести ФИО)
* При тестировании "боевого сайта" возможно нужно контролировать скорость выполнения тестов, чтобы избежать "перегрузки"
